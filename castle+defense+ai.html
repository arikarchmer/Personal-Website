Can You Rule Riddler Nation - FiveThirtyEight Riddle¶

Initialization and Comparison¶

First we have a function named init_pop(n) which will allows us to randomely generate n viable strategies. Additionally we initialize each strategies score to 0.

The fight(s1, s2) method serves as a function to compare to strategies. Fight(s1, 2s) returns a list with a 1 for the winning strategy, and a 0 for the losing strategy. The logic in this function is as specified by FiveThirtyEight's riddle.

DESIGN DECISION:
The question of generating random strategies is not trivial. How can we quickly fill a list with 100 (or near 100) total soldiers? Of course we can pick random numbers, but in order to complete this task in just one pass through of the ten castles, we must keep track of how many soldiers we have deployed as we go. Thus, picking a random number ranging from 0 to the remaining number of available soldiers may skew the stratgey to having more soldiers towards the beginning. Thus, I have implemented two design decisions to help comabt this. First, the random number of soldiers is chosen from 0 to the remaining available / 2. This prevents the random number from being too high, and depriving many castles of soldiers later on. Second, The first castle considered is the most valuable castle. This decision will hopefully reflect the idea that many participants in the competition will likely send a disproportionately high number of soldiers to the more valuvable castles.

import random

def init_pop(n):
    
    strats = []
    for i in range(n):
        sols = 100
        castles = []
        for t in range(10):
            delegation = random.randint(0, sols / 2)
            castles.insert(0, delegation)
            sols -= delegation
        strats.append([castles, 0])
    return strats
    
def fight(s1, s2):
    s1 = s1[0]
    s2 = s2[0]
    s1score = 0
    s2score = 0
    for i in range(10):
        if s1[i] > s2[i]:
            s1score += i + 1
        if s1[i] < s2[i]:
            s2score += i + 1
            
#     return [s1score, s2score]
            
    if s1score > s2score:
        return [1 , 0]
    elif s1score < s2score:
        return [0, 1]
    else:
        return [0.5, 0.5]

Strategy Fitness (Rankings)¶

Next, we define a function compute_scores(population) that will allow us to simulate a round robin style tournament between all our current candidate strategies. This method returns a sorted list of the stretegies with the highest performing ones at the beginning, where the first element of the list is the strategy, and the second is the number of wins this strategy acuired in the round robin simulation.

def compute_scores(population):
    
    #re init scores
    for i in range(len(population)):
        population[i][1] = 0

    for i in range(len(population)):
        for i2 in range(i+1, len(population)):
            
            res = fight(population[i], population[i2])
            population[i][1] += res[0]
            population[i2][1] += res[1]
            
    population = sorted(population, key=lambda x: x[1], reverse=True)
    
    return population

Genetic Algorithm¶

This is where the magic happens. We define functions evolve(strats), mate(t1, t1), check_constraints(strat), and mating(pack). These functions will collectively allow us to implement a genetic algorithm to find near optimal strategies for the riddle. During each iteration, the top 10% strategies are designated as 'elite,' and persist through generations. Additionally, the bottom 20% drop out after each generation, and 10% new strategies are added to the population. The rest of the strategies, known as the pack, use the mate(t1, t2) function to produce a child who will join the pack as a new candidate in the next generation.

def mate(t1, t2):
    
    new = []
    r = 0
    for i in range(10):
        r += 1
        if r % 2 == 0:
            new.append(t1[0][i])
        else:
            new.append(t2[0][i])
    return new


def check_constraints(strat):
    s = 0
    for i in strat:
        s += i
    if s > 100 or s < 80:
        return False
    return True


def mating(pack):
    new_teams = []
    i = 0

    packlen = len(pack)
    while i <= (packlen - 1):
        possible = mate(pack[i], pack[random.randint(i / 2, i)])
        if check_constraints(possible):
            possible_w_score = [possible, 0]
            new_teams.append(possible_w_score)

            i += 1
    
    return new_teams

    return [top_team_per_gen, w_percs]

DESIGN DECISION:
The parameters of an initial population of 10000 with 25 generations were chosen because I believe that it is particularly important to include a wide variety of viable strategies to test against. 25 generations were chosen in an interest of time since the population was chosen to be so large. I believe this type of setup is more effective than one with say a population of 1000 and 100 generations because we would like to eliminate the chance that one strategy may perform particularly well against 1000 others, but not against different types (overfitting.) Having 10000 strategies in the population forces successful strategies to be effective against a very wide variety of other strategies.
